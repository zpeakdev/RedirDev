# Chrome 扩展工程化开发规范（基于 Manifest V3）

## 一、目录规范

### 1.1 标准项目结构

```text
my-chrome-extension/
├── .eslintrc.json          # ESLint 配置
├── .prettierrc             # Prettier 配置
├── package.json            # 项目依赖管理
├── README.md               # 项目说明
├── build/                  # 构建输出目录
├── dist/                   # 打包输出目录
├── src/
│   ├── background/         # 后台脚本
│   │   └── main.js         # 主后台脚本
│   ├── content/            # 内容脚本
│   │   ├── content.js      # 主内容脚本
│   │   └── content.css     # 内容样式
│   ├── popup/              # 弹出页面
│   │   ├── popup.html      # 弹出页面
│   │   ├── popup.js        # 弹出逻辑
│   │   └── popup.css       # 弹出样式
│   ├── icons/              # 图标资源
│   │   ├── 16.png          # 16x16 图标
│   │   ├── 48.png          # 48x48 图标
│   │   └── 128.png         # 128x128 图标
│   └── utils/              # 工具函数
│       └── api.js          # API 封装
├── manifest.json           # 扩展配置文件
└── styles/                 # 全局样式
    └── main.css            # 全局样式
```

### 1.2 目录规范说明

1. **src/** - 源代码目录，所有开发文件存放于此
2. **build/** - 构建临时文件
3. **dist/** - 最终打包输出目录
4. **icons/** - 必须包含 16x16、48x48、128x128 三种尺寸的图标
5. **background/** - 后台脚本，使用 Service Worker 模式
6. **content/** - 内容脚本，用于注入网页
7. **popup/** - 弹出页面，包含 HTML、JS、CSS
8. **utils/** - 工具函数，避免重复代码

## 二、manifest.json 规范

### 2.1 基础配置

```json
{
  "manifest_version": 3,
  "name": "扩展名称",
  "version": "1.0.0",
  "description": "扩展描述（不超过132字符）",
  "author": "作者名称",
  "homepage_url": "https://example.com",
  "icons": {
    "16": "icons/16.png",
    "48": "icons/48.png",
    "128": "icons/128.png"
  },
  "action": {
    "default_popup": "popup/popup.html",
    "default_icon": {
      "16": "icons/16.png",
      "48": "icons/48.png"
    }
  },
  "background": {
    "service_worker": "background/main.js",
    "type": "module"
  },
  "permissions": ["storage", "activeTab", "scripting"],
  "host_permissions": ["https://*.example.com/*", "https://example.com/*"],
  "content_scripts": [
    {
      "matches": ["<all_urls>"],
      "js": ["content/content.js"],
      "css": ["content/content.css"],
      "run_at": "document_idle"
    }
  ],
  "default_locale": "zh-CN",
  "content_security_policy": {
    "extension_pages": "script-src 'self'; object-src 'self'",
    "sandbox": "sandbox allow-scripts; script-src 'self'; object-src 'self'"
  }
}
```

### 2.2 关键规范

1. **manifest_version 必须为 3**，禁止使用 2
2. **name** 长度 ≤ 45 字符
3. **description** 长度 ≤ 132 字符
4. **version** 采用语义化版本（x.y.z）
5. **icons** 必须包含 16/48/128 像素
6. **permissions** 仅包含必需权限，避免使用 `<all_urls>`
7. **host_permissions** 必须精确指定域名，避免使用通配符
8. **content_scripts** 的 `run_at` 必须指定为 `document_idle` 或 `document_start`
9. **background** 必须使用 `service_worker` 而非 `scripts`
10. **content_security_policy** 必须配置，避免使用 `unsafe-eval`

## 三、代码规范

### 3.1 代码风格

1. **使用 ESLint + Prettier** 统一代码风格
2. **ES6+ 语法**：使用 `const`/`let`、箭头函数、模块导入等
   - 必须使用 `ts`实现功能逻辑
3. **命名规范**：
   - 文件名：`kebab-case`（如 `content-script.js`）
   - 变量名：`camelCase`（如 `contentScript`）
   - 函数名：`camelCase`（如 `getContent`）
   - 类名：`PascalCase`（如 `ContentScript`）
4. **注释规范**：
   - 每个函数必须有 JSDoc 注释
   - 复杂逻辑必须添加注释

### 3.2 代码示例

**background/main.js**:

```javascript
/**
 * 后台脚本 - 处理扩展核心逻辑
 * @module background
 */

import { storage } from "chrome";

/**
 * 初始化后台逻辑
 */
export function init() {
  console.log("扩展已启动");

  // 监听消息
  chrome.runtime.onMessage.addListener((request, sender, sendResponse) => {
    if (request.action === "getSettings") {
      storage.local.get(["theme"], (result) => {
        sendResponse(result);
      });
      return true; // 表示异步响应
    }
  });
}

// 初始化
init();
```

**popup/popup.js**:

```javascript
/**
 * 弹出页面逻辑
 * @module popup
 */

/**
 * 初始化弹出页面
 */
function init() {
  const button = document.getElementById("toggleButton");
  const message = document.getElementById("message");

  button.addEventListener("click", () => {
    chrome.tabs.query({ active: true, currentWindow: true }, (tabs) => {
      chrome.scripting.executeScript(
        {
          target: { tabId: tabs[0].id },
          files: ["content/content.js"],
        },
        () => {
          message.textContent = "内容脚本已注入";
        },
      );
    });
  });
}

// 页面加载完成后初始化
document.addEventListener("DOMContentLoaded", init);
```

## 四、安全与权限规范

### 4.1 权限最小化原则

1. **仅请求必要权限**，避免请求过多权限
2. **避免使用 `<all_urls>`**，改用精确匹配
3. **`host_permissions` 仅包含必需域名**
4. **`permissions` 仅包含必需功能权限**

### 4.2 安全配置

1. **必须配置 `content_security_policy`**
2. **禁止使用 `eval` 和 `unsafe-eval`**
3. **避免在扩展中存储敏感信息**
4. **使用 `chrome.storage` 而非 `localStorage`**
5. **对用户数据进行加密存储**

## 五、构建与发布规范

### 5.1 构建流程

1. **开发阶段**：
   - 使用 `npm run dev` 启动开发服务器
   - 使用 `npm run build` 构建生产版本
   - 使用 `npm run lint` 检查代码

2. **构建配置**：

   ```json
   {
     "scripts": {
       "dev": "webpack serve --config webpack.config.js",
       "build": "webpack --config webpack.config.js",
       "lint": "eslint src --ext js,jsx"
     }
   }
   ```

### 5.2 发布流程

1. **验证**：
   - 通过 `chrome://extensions/` 加载未打包扩展
   - 测试所有功能
   - 运行 `npm run lint` 检查代码

2. **打包**：

   ```bash
   npm run build
   ```

3. **提交应用商店**：
   - 压缩 `dist/` 目录为 zip
   - 提交到 Chrome 应用商店
   - 提供清晰的描述和截图

## 六、测试与调试规范

### 6.1 测试规范

1. **单元测试**：
   - 使用 vitest 对核心逻辑进行测试
   - 测试后台脚本、内容脚本、工具函数

2. **功能测试**：
   - 测试所有功能场景
   - 测试不同浏览器版本的兼容性
   - 测试不同 URL 域名的兼容性

### 6.2 调试技巧

1. **开发者模式**：
   - 启用 `chrome://extensions/` 中的 "开发者模式"
   - 使用 "加载已解压的扩展程序" 加载开发版

2. **调试工具**：
   - **背景脚本**：`chrome://extensions/` 中的 "服务工作者" 选项
   - **内容脚本**：在目标网页中打开开发者工具
   - **弹出页面**：在扩展页面中右键 "检查" 打开开发者工具

3. **日志规范**：
   - 使用 `console.log` 记录关键信息
   - 避免在生产环境中使用 `console.log`
   - 使用 `chrome.runtime.sendMessage` 发送日志

## 七、进阶规范

### 7.1 多语言支持

1. 在 `src/locales/` 目录下创建多语言文件
2. 在 `manifest.json` 中配置 `default_locale`
3. 使用 `chrome.i18n.getMessage` 获取本地化字符串

### 7.2 性能优化

1. **后台脚本**：
   - 使用 `chrome.alarms` 替代定时器
   - 避免长时间运行的脚本

2. **内容脚本**：
   - 仅在必要时注入
   - 使用 `run_at: "document_idle"` 提升性能

3. **资源加载**：
   - 压缩图片资源
   - 使用 CDN 加载外部资源（如果必要）

## 八、常见错误与解决方案

| 问题                  | 原因                            | 解决方案              |
| --------------------- | ------------------------------- | --------------------- |
| 扩展无法加载          | manifest_version 为 2           | 改为 3                |
| 弹出页面不显示        | default_popup 路径错误          | 检查路径是否正确      |
| 内容脚本无法注入      | permissions 缺少 scripting      | 添加 "scripting" 权限 |
| 网络请求失败          | host_permissions 未配置         | 添加所需域名          |
| 安全错误              | content_security_policy 未配置  | 添加正确的 CSP 配置   |
| 无法访问 localStorage | 未在 permissions 中添加 storage | 添加 "storage" 权限   |

## 九、总结

这套工程化规范基于 Manifest V3 的最佳实践，旨在确保 Chrome 扩展的：

- **安全性**：最小权限原则，严格的安全配置
- **可维护性**：清晰的目录结构，标准化的代码风格
- **可扩展性**：模块化设计，便于功能扩展
- **可发布性**：符合 Chrome 应用商店的规范要求

遵循这些规范，可以显著提高 Chrome 扩展的质量，减少审核问题，并提升用户体验。
