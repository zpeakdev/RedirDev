manifest v3实现网络请求代理功能的方案



在 Manifest V3（MV3）中，Chrome 出于性能和安全考虑，**移除了 `webRequest` 的 `blocking` 模式**（仅保留给少数特权扩展或企业策略）。Service Worker 的异步事件驱动模型也决定了无法再同步拦截/修改请求。因此，MV3 实现网络请求代理/拦截需采用新方案。

以下是目前主流且官方推荐的实现路径，按场景分类说明：

---
## 📊 方案对比速览
| 方案                          | 能否阻断/修改请求                | 动态性               | 性能  | 适用场景                                 |
| ----------------------------- | -------------------------------- | -------------------- | ----- | ---------------------------------------- |
| `declarativeNetRequest` (DNR) | ✅ 支持规则级修改/重定向/改Header | ⚠️ 动态规则有数量限制 | ⭐⭐⭐⭐⭐ | 广告拦截、请求转发、Header注入、跨域处理 |
| `chrome.proxy`                | ✅ 全流量路由代理                 | ✅ 高（PAC/动态切换） | ⭐⭐⭐   | 全局代理、抓包工具、企业网络代理         |
| `webRequest`（非阻塞）        | ❌ 仅监听/分析                    | ✅ 实时               | ⭐⭐⭐⭐  | 请求调试、流量统计、触发业务逻辑         |

---
## 🛠 方案一：`declarativeNetRequest`（推荐，绝大多数场景）
基于规则匹配，由浏览器底层直接执行，Service Worker 休眠不影响规则生效。

### 1. 权限配置 `manifest.json`
```json
{
  "manifest_version": 3,
  "name": "MV3 Request Proxy",
  "version": "1.0",
  "permissions": [
    "declarativeNetRequest"
  ],
  "host_permissions": [
    "<all_urls>"
  ],
  "declarative_net_request": {
    "rule_resources": [
      {
        "id": "ruleset_1",
        "enabled": true,
        "path": "rules.json"
      }
    ]
  }
}
```

### 2. 静态规则 `rules.json`
```json
[
  {
    "id": 1,
    "priority": 1,
    "action": {
      "type": "redirect",
      "redirect": { "transform": { "host": "mock.api.com" } }
    },
    "condition": {
      "urlFilter": "api.example.com/*",
      "resourceTypes": ["xmlhttprequest", "fetch"]
    }
  },
  {
    "id": 2,
    "action": {
      "type": "modifyHeaders",
      "requestHeaders": [
        { "header": "X-Custom-Header", "operation": "set", "value": "mv3-proxy" }
      ]
    },
    "condition": {
      "urlFilter": "*.example.com/*",
      "resourceTypes": ["main_frame", "sub_frame", "xmlhttprequest"]
    }
  }
]
```

### 3. 动态规则管理（运行时增删）
```javascript
// 添加动态规则
chrome.declarativeNetRequest.updateDynamicRules({
  addRules: [
    {
      id: 100,
      priority: 2,
      action: { type: "block" },
      condition: {
        urlFilter: "||tracker.example.com^",
        resourceTypes: ["script", "image"]
      }
    }
  ],
  removeRuleIds: [100] // 移除旧规则
});

// 查询规则命中统计（需声明 declarativeNetRequestFeedback 权限）
chrome.declarativeNetRequest.getMatchedRulesDetails({}).then(details => {
  console.log(details.rulesMatchedInfo);
});
```

📌 **注意**：
- 动态规则默认上限 `30,000` 条（可通过 `chrome.declarativeNetRequest.MAX_NUMBER_OF_DYNAMIC_RULES` 查询）
- 规则匹配使用 **过滤语法**（类似 Adblock Plus），支持通配符、正则（需开启 `isRegex`）
- 不支持基于响应体动态修改请求（架构限制）

## 🌐 方案二：`chrome.proxy`（完整网络代理）
适用于需要将所有/部分流量转发到代理服务器（如 HTTP/SOCKS5/PAC）的场景。

### 权限与配置
```json
{
  "permissions": ["proxy", "tabs"],
  "host_permissions": ["<all_urls>"]
}
```

### 代码示例
```javascript
// 设置固定代理
chrome.proxy.settings.set({
  value: {
    mode: "fixed_servers",
    rules: {
      singleProxy: { scheme: "http", host: "127.0.0.1", port: 8080 },
      bypassList: ["localhost", "127.0.0.1"]
    }
  },
  scope: "regular"
});

// 或使用 PAC 脚本
chrome.proxy.settings.set({
  value: { mode: "pac_script", rules: { data: `function FindProxyForURL(url, host) { return "PROXY 127.0.0.1:8080"; }` } },
  scope: "regular"
});
```

📌 **注意**：
- 影响系统级代理栈，用户会看到代理切换提示
- 无法按请求精细修改 Header 或阻断特定 URL（需配合 DNR）
- 适合抓包工具、企业网络代理、全局翻墙插件

## 📡 方案三：`webRequest`（仅监听/分析）
MV3 仍保留该 API，但**移除 `blocking` 修饰符**，只能异步监听。

```javascript
chrome.webRequest.onBeforeRequest.addListener(
  (details) => {
    console.log("拦截到请求:", details.url);
    // 可触发日志、统计、通知其他模块，但不能 return { cancel: true }
  },
  { urls: ["<all_urls>"] }
);
```
📌 **适用场景**：流量分析、调试面板、行为埋点。**不能用于代理/拦截**。

## 🔧 高级场景处理技巧
| 需求                     | MV3 实现方案                                                 |
| ------------------------ | ------------------------------------------------------------ |
| 动态规则过多             | 合并规则、使用正则过滤、按域名分组加载                       |
| 需根据页面上下文修改请求 | Content Script 获取上下文 → `chrome.runtime.sendMessage` → 动态更新 DNR 规则 |
| 规则冲突处理             | 使用 `priority` 字段（值越大优先级越高）                     |
| 调试规则命中             | 开启 `declarativeNetRequestFeedback` 权限，访问 `chrome://extensions` 查看详情 |
| 跨域/重定向限制          | 浏览器安全策略仍生效，DNR 无法绕过 CORS/CSP                  |

## ⚠️ 重要限制与建议
1. **不要尝试用 MV2 思维写 MV3**：`blocking webRequest` 已彻底废弃，普通扩展无法绕过
2. **规则持久化**：DNR 规则由浏览器内核管理，Service Worker 休眠不影响生效
3. **性能优化**：避免使用宽泛的 `"<all_urls>"` 条件，尽量缩小 `urlFilter` 范围
4. **审核注意**：修改请求 Header 或重定向可能触发 Chrome 安全策略，需提供明确用途说明
5. **未来趋势**：Chrome 正逐步增强 DNR 能力（如正则性能、更细粒度 `resourceTypes`），建议以 DNR 为核心架构

---
## ✅ 选型建议
- **广告拦截/请求过滤/改Header** → `declarativeNetRequest`
- **全局代理/抓包转发** → `chrome.proxy` + `declarativeNetRequest`（组合使用）
- **仅做请求监控/调试** → `webRequest`（非阻塞）
- **需基于响应体动态修改请求** → 目前 MV3 无原生支持，需改用后端代理或调整业务架构

### 2. `extension/src/background.js` (MV3 Service Worker)
```javascript
const DEFAULT_PORT = "8888";

chrome.runtime.onInstalled.addListener(() => {
  chrome.storage.local.get(['proxyEnabled', 'targetPattern', 'proxyPort'], (data) => {
    if (data.proxyEnabled) {
      setProxy(data.targetPattern || 'api.example.com', data.proxyPort || DEFAULT_PORT);
    }
  });
});

chrome.runtime.onMessage.addListener((msg, _, sendResponse) => {
  if (msg.action === 'TOGGLE_PROXY') {
    if (msg.enabled) {
      setProxy(msg.targetPattern, msg.port);
      chrome.storage.local.set({ proxyEnabled: true, targetPattern: msg.targetPattern, proxyPort: msg.port });
    } else {
      clearProxy();
      chrome.storage.local.set({ proxyEnabled: false });
    }
    sendResponse({ status: 'ok' });
  }
  return true; // 保持消息通道开放
});

function setProxy(pattern, port) {
  const pacScript = `
    function FindProxyForURL(url, host) {
      if (url.includes("${pattern}")) return "PROXY 127.0.0.1:${port}";
      return "DIRECT";
    }
  `;
  chrome.proxy.settings.set({
    value: { mode: "pac_script", rules: { data: pacScript } },
    scope: "regular"
  });
}

function clearProxy() {
  chrome.proxy.settings.clear({ scope: "regular" });
}
```

export default function App() {

### 4. `proxy-server/index.js` (本地代理服务)
```bash
# 安装依赖
npm init -y && npm install http-proxy cors
```
```javascript
const http = require('http');
const httpProxy = require('http-proxy');

const proxy = httpProxy.createProxyServer();
const LOCAL_TARGET = 'http://localhost:3000'; // 你的本地开发服务地址

const server = http.createServer((req, res) => {
  // 1. 缓存请求体（流必须消费完才能修改）
  let body = [];
  req.on('data', chunk => body.push(chunk));
  req.on('end', () => {
    let raw = Buffer.concat(body).toString();
    let parsed = safeParse(raw);

    // 2. 修改请求方法 & 请求头
    req.method = 'POST';
    req.headers['x-proxy-intercepted'] = 'true';
    req.headers['content-type'] = 'application/json';

    // 3. 修改请求参数
    parsed._mocked = true;
    parsed._timestamp = Date.now();
    req.headers['content-length'] = Buffer.byteLength(JSON.stringify(parsed));

    // 4. 转发到本地服务（selfHandleResponse: true 允许我们拦截响应）
    proxy.web(req, res, { target: LOCAL_TARGET, selfHandleResponse: true, changeOrigin: true });
  });
});

// 5. 拦截并修改响应数据
proxy.on('proxyRes', (proxyRes, req, res) => {
  let body = [];
  proxyRes.on('data', chunk => body.push(chunk));
  proxyRes.on('end', () => {
    let raw = Buffer.concat(body).toString();
    try {
      let json = safeParse(raw);
      json._proxyModified = true;
      json.data = { mock: "本地代理返回的数据", original: json.data };
      raw = JSON.stringify(json);
    } catch {}

    res.writeHead(proxyRes.statusCode, {
      'Content-Type': 'application/json',
      'Access-Control-Allow-Origin': '*',   // 关键：解决 CORS
      'Access-Control-Allow-Methods': 'GET,POST,PUT,DELETE,OPTIONS',
      'Access-Control-Allow-Headers': 'Content-Type,Authorization',
      'X-Proxy-Modified': 'true'
    });
    res.end(raw);
  });
});

// 处理 CORS 预检请求
server.on('request', (req, res) => {
  if (req.method === 'OPTIONS') {
    res.writeHead(204, {
      'Access-Control-Allow-Origin': '*',
      'Access-Control-Allow-Methods': 'GET,POST,PUT,DELETE,OPTIONS',
      'Access-Control-Allow-Headers': 'Content-Type,Authorization'
    });
    res.end();
  }
});

function safeParse(str) {
  try { return JSON.parse(str || '{}'); } catch { return {}; }
}

server.listen(8888, () => console.log('✅ 代理服务器已启动: http://127.0.0.1:8888'));
```

## ⚠️ MV3 特有避坑指南
| 问题                                         | 原因                                          | 解决方案                                                     |
| -------------------------------------------- | --------------------------------------------- | ------------------------------------------------------------ |
| **页面报 CORS 错误**                         | 浏览器跨域策略在代理返回后生效                | 本地服务必须返回 `Access-Control-Allow-Origin`（代码已内置） |
| **HTTPS 页面请求 `http://localhost` 被拦截** | Chrome 默认阻止混合内容                       | 开发期访问 `chrome://flags` 开启 `#allow-insecure-localhost`，或为本地服务配 HTTPS 证书 |
| **Service Worker 休眠导致代理失效**          | `chrome.proxy` 是系统级设置，不受 SW 休眠影响 | ✅ 已规避。PAC 由浏览器内核持久管理                           |
| **请求体丢失**                               | Node `req` 是流，未消费直接转发会清空         | 代码已用 `req.on('data')` 缓存完整 Body                      |
| **扩展无法打包本地服务**                     | Chrome 安全沙箱禁止扩展启动进程               | ✅ 架构已分离。扩展管路由，Node 管拦截，符合现代 DevTool 范式（如 Whistle/Charles） |

## 📦 核心架构（纯扩展内实现）
```
[页面请求] → [Content Script 拦截 fetch/XHR] → [匹配规则] → [修改方法/头/参数]
       ↓
   [代理到 http://127.0.0.1:你的本地端口] 或 [直接返回 Mock 响应]
       ↓
[拦截响应数据] → [修改响应体] → [返回给页面]
```
✅ **优势**：无需额外启动 Node 代理进程、安装即用、完全兼容现代前端工作流。

## 🛠 完整代码实现

### 1. `manifest.json`
```json
{
  "manifest_version": 3,
  "name": "API Proxy & Mock",
  "version": "1.0.0",
  "permissions": ["storage", "scripting"],
  "host_permissions": ["<all_urls>"],
  "background": { "service_worker": "background.js" },
  "action": { "default_popup": "popup/index.html" },
  "content_scripts": [
    {
      "matches": ["<all_urls>"],
      "js": ["content.js"],

      "world": "MAIN"
    }

```
📌 `world: "MAIN"` 确保脚本运行在页面主上下文，可拦截 `window.fetch` 和 `XMLHttpRequest`。

### 2. `content.js`（核心拦截引擎）
```javascript
// content.js
const ORIGINAL_FETCH = window.fetch;
const ORIGINAL_XHR = window.XMLHttpRequest;

let rules = [];

// 从 background 同步规则
chrome.runtime.onMessage.addListener((msg) => {
  if (msg.type === 'UPDATE_RULES') rules = msg.rules;
});

// 初始加载
chrome.runtime.sendMessage({ type: 'GET_RULES' }, (res) => {
  if (res && res.rules) rules = res.rules;
});

// 辅助：安全解析 JSON
const safeParse = (str) => {
  try { return JSON.parse(str); } catch { return {}; }
};

// 匹配规则
const matchRule = (url) => rules.find(r => url.includes(r.match)) || null;

// ==================== FETCH 拦截 ====================
window.fetch = async function(input, init = {}) {
  const url = typeof input === 'string' ? input : input.url;
  const rule = matchRule(url);
  if (!rule) return ORIGINAL_FETCH.apply(this, arguments);

  // 1. 构建新请求配置
  const newInit = { ...init, method: rule.method || init.method || 'GET' };
  newInit.headers = new Headers(init.headers || {});
  Object.entries(rule.headers || {}).forEach(([k, v]) => newInit.headers.set(k, v));

  // 2. 修改请求参数/Body
  if (init.body && rule.modifyBody) {
    let bodyStr = typeof init.body === 'string' ? init.body : init.body.toString();
    let bodyObj = safeParse(bodyStr);
    let newBody = rule.modifyBody(bodyObj);
    newInit.body = JSON.stringify(newBody);
    newInit.headers.set('Content-Type', 'application/json');
  }

  // 3. 确定目标地址
  let targetUrl = url;
  if (rule.proxyToLocal) {
    const port = rule.localPort || 3000;
    targetUrl = `http://127.0.0.1:${port}${new URL(url).pathname}`;
  }

  // 4. 发起请求
  let res;
  try {
    res = await ORIGINAL_FETCH(targetUrl, newInit);
  } catch (err) {
    console.warn(`[Proxy] Fetch failed for ${url}`, err);
    return new Response(JSON.stringify({ error: 'Proxy failed' }), { status: 502 });
  }

  // 5. 修改响应数据
  if (rule.modifyResponse) {
    const cloned = res.clone();
    const data = await cloned.json().catch(() => ({}));
    const newData = rule.modifyResponse(data);
    return new Response(JSON.stringify(newData), {
      status: rule.status || res.status,
      statusText: res.statusText,
      headers: res.headers
    });
  }

  return res;
};

// ==================== XHR 拦截（简化核心逻辑） ====================
const origOpen = ORIGINAL_XHR.prototype.open;
const origSend = ORIGINAL_XHR.prototype.send;

XMLHttpRequest.prototype.open = function(method, url, ...args) {
  this._proxyRule = matchRule(url);
  this._proxyMethod = method;
  this._proxyUrl = url;
  return origOpen.call(this, method, url, ...args);
};

XMLHttpRequest.prototype.send = function(body) {
  const rule = this._proxyRule;
  if (!rule) return origSend.call(this, body);

  // 修改方法
  const method = rule.method || this._proxyMethod;
  origOpen.call(this, method, this._proxyUrl, true, this._proxyArgs?.[2], this._proxyArgs?.[3]);

  // 修改头
  if (rule.headers) {
    Object.entries(rule.headers).forEach(([k, v]) => this.setRequestHeader(k, v));
  }

  // 修改 Body
  let finalBody = body;
  if (body && rule.modifyBody) {
    const obj = safeParse(body);
    finalBody = JSON.stringify(rule.modifyBody(obj));
  }

  // 重写响应拦截
  const self = this;
  this._origReadyStateChange = this.onreadystatechange;
  this.onreadystatechange = function() {
    if (self.readyState === 4 && rule.modifyResponse) {
      try {
        const originalData = safeParse(self.responseText);
        const newData = rule.modifyResponse(originalData);
        Object.defineProperty(self, 'responseText', { get: () => JSON.stringify(newData) });
        Object.defineProperty(self, 'response', { get: () => newData });
      } catch {}
    }
    if (self._origReadyStateChange) self._origReadyStateChange.call(self);
  };

  // 代理到本地或直连
  const targetUrl = rule.proxyToLocal
    ? `http://127.0.0.1:${rule.localPort || 3000}${new URL(this._proxyUrl).pathname}`
    : this._proxyUrl;

  return origSend.call(this, finalBody);
};
```

const DEFAULT_RULES = [
  {
    id: 1,
    match: 'api.example.com/v1/user',

  }

chrome.runtime.onInstalled.addListener(() => {
  chrome.storage.local.get('proxyRules', (data) => {
    if (!data.proxyRules) chrome.storage.local.set({ proxyRules: DEFAULT_RULES });
  });
});

chrome.runtime.onMessage.addListener((msg, sender, sendResponse) => {
  if (msg.type === 'GET_RULES') {
    chrome.storage.local.get('proxyRules', (data) => sendResponse({ rules: data.proxyRules || [] }));
    return true;
  }
  if (msg.type === 'UPDATE_RULES') {
    chrome.storage.local.set({ proxyRules: msg.rules });
    // 广播给所有已注入的 Content Script
    chrome.tabs.query({}, (tabs) => {
      tabs.forEach(tab => {
        chrome.tabs.sendMessage(tab.id, { type: 'UPDATE_RULES', rules: msg.rules }, () => {});
      });
    });
    sendResponse({ status: 'ok' });
    return true;
  }
});

## 🔑 关键机制说明

| 需求                     | 实现方式                                                     |
| ------------------------ | ------------------------------------------------------------ |
| **安装即用**             | 无外部依赖。Content Script 随页面加载自动注入，拦截逻辑在页面主线程执行 |
| **修改请求方法/头/参数** | 拦截 `fetch`/`XHR` → 读取规则 → 覆盖 `method`/`headers` → 序列化修改后的 Body 重新发送 |
| **代理到本地服务**       | `proxyToLocal: true` 时，自动将 URL 路径重写为 `http://127.0.0.1:{localPort}{pathname}`，**直接对接你已有的 Vite/Node 服务** |
| **修改响应数据**         | `res.clone()` 安全读取响应体 → 执行 `modifyResponse` 转换函数 → 返回全新 `Response` 对象 |
| **动态生效**             | `chrome.storage` 持久化规则 + `chrome.runtime.onMessage` 广播，修改后立即刷新页面试用 |

## ⚠️ MV3 特有避坑指南

1. **CORS 问题**：扩展拦截发生在浏览器发送请求**之前**，因此不会触发跨域限制。本地服务无需额外配置 CORS。
2. **Body 流只能消费一次**：代码中使用 `JSON.stringify()` 重新序列化，避免 `ReadableStream` 被清空导致后续请求失败。
3. **Vite 本地服务配合**：确保你的本地服务监听 `127.0.0.1`（非 `0.0.0.0`），否则 Chrome 可能拒绝连接。
4. **HTTPS 页面请求 HTTP 本地服务**：Chrome 默认拦截混合内容。开发期访问 `chrome://flags` 开启 `#allow-insecure-localhost`，或为你的本地服务配置 HTTPS。
5. **Service Worker 休眠**：拦截逻辑在 Content Script 中运行，**完全不受 SW 休眠影响**，页面刷新后规则自动同步。
